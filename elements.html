Petro Chemical Manufacturer in Saudi Arabia

Aug 2020 - Jan 2021

Problem Statement: Client faces different slippage incident or outages in many Towers of manufacturing plant. Incident leads to shutdown of equipment’s or entire tower or part of the plant. It causes financial loss due to off specification of output from the Tower. Client wanted an early warning system to predict the slippage incident using past data.
Project: To predict the incidents early to avoid outage or shutdown of the equipment or entire Tower.
Kind of Data: IOT sensors collect data about the chemical flow, level, temperature, pressure and many metrics.
Data Extraction: Data is captured every second; we aggregated data at minute level.
Data processing: Handled missing data, dropped redundant variables, added different date variables, lag variables.
Problem type: Regression problem to predict Incident (Anomaly incident) using past data.
Approach: Implemented XG Boost model for good accuracy and revamped Ridge regression as explanatory model.
Outcome: Plant operator will get alert in dashboard about the incident 4 hour in advance and 1 hour inadvance. Operator can adjust input feed using the influencing variables from explanatory model.
Use-cases worked on:
• Predicted Slippage of Acid gases (CO2 and H2S) in Caustic Tower/Cleaning Tower.
• Predicted Slippage of NOx and Excess Air in the Furnace towers.
• Predicted Stack Temperature in Furnace Tower/Heating Tower and Overhead temperature in Quench Tower/Cooling Tower.
• End to End Data science Life cycle.
• Requirement Gathering, Problem understanding, Data extraction, Data preprocessing, Data modelling, Model Hyperparameter tuning, Model deployment.

Language Used: Python, SQL
Tools Used: Dataiku

Largest technology supplier for Food processing and other industries, Germany

June 2021 - Oct 2021

Problem Statement: Client has 208 business units and it is important to forecast many metrics like Order Intake, Revenue, Production planning, etc.,
Project: To build framework to forecast any business metrics to increase the quality of sales.
Kind of Data: Historical data of metrics like Order Intake and revenue.
Data Extraction: Loaded data to SQL database on azure. Used Azure JDBC connections to fetch data from SQL database to Azure data bricks notebooks.
Data processing: Considered complete records only; Date is converted to required format, Business units which has less than 2 data points are filtered out.

Stock Tweets Prediction

A hedge fund uses 6 financial factors to predict the alpha signal in a stock. This alpha signal is used to make purchase decisions about the stock. The hedge fund now collected and tagged microblogging data for sentiment from the Social Media platform called ‘StockTwits’.StockTwits is used by people who regularly trade stocks. People on this platform tweet about stocks using the special character ‘$’ to indicate the name of the stock. These microblogs similar to tweets might contain important information about the alpha signal in a stock.Your goal is to build a sentiment analysis model using the tagged data. This sentiment analysis model should then be used to generate a new stock factor which together with the other stock factors should be used to predict the Alpha Signal.The hedge fund has anonymised the data, which contains 7 stock factors and an alpha signal. This alpha signal is generated using a near perfect algorithmic trading strategy. Unfortunately the number of stock factors, collected to run that strategy,are extremely high and have to be collected from a large number of data vendors at a high price.
